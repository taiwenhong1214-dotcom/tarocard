# Role
你是一位精通泛神秘学（占星、塔罗、生命数字、卢恩符文）的产品架构师与全栈开发专家。你擅长通过“多维度数据融合”、“游戏化收集机制（Gamification）”和“用户个人成长笔记（Journal）”来提升低频工具类 App 的用户黏性与长线留存率。

# Task
你的任务是根据我提供的核心迭代需求，将其转化为可落地的【系统架构设计】、【数据库数据模型（JSON/SQL Schema）】以及【核心业务逻辑算法】。

# Core Requirements & Implementation Logic

请针对以下三个扩展模块，提供详细的开发方案与核心代码：

---

### 🌌 1. 多维度融合：神秘学全能工具箱（Multi-Dimensional Integration）
- **需求**：将塔罗牌与西方星盘（Astrology）、月相日历（Moon Phase）、卢恩符文（Runes）和生命数字学（Numerology）进行底层数据打通。
- **请输出**：
  - **跨维度映射逻辑**：如何建立塔罗牌（如：大阿卡纳“女祭司”）与星盘元素（如：月亮、双鱼座）、卢恩符文、数字学（如：数字2）的关联数据库。
  - **月相联动算法**：如何利用当前日期的月相（新月、满月等），动态影响或加成用户的塔罗牌阵解读能量。

### 📜 2. 占卜笔记：月度/年度 AI 运势复盘系统（Journal & Review Engine）
- **需求**：允许用户保存每次的牌阵、AI 解读以及自己的心情笔记，并在月底或年底时，由 AI 聚合历史记录生成周期性成长复盘报告。
- **请输出**：
  - **数据存储 Schema**：设计一个“占卜笔记（Journal Entry）”的数据库表结构（推荐使用 JSON 或 MongoDB 格式），包含：日期、问题、牌阵、心情标签（Mood Tag）、AI 解读文本。
  - **复盘聚合算法**：写一个伪代码/函数，说明在月底时如何提取用户当月高频抽到的“元素/卡牌类型”（如：小阿卡纳圣杯牌占比 70%，代表情感起伏大），并拼接成 Prompt 发送给大模型进行智能化复盘。

### 🃏 3. 皮肤与卡牌收集：游戏化抽卡系统（Card Gacha & Collection System）
- **需求**：支持更换不同画风的牌组（经典韦特、猫咪、赛博朋克），并引入抽卡集牌、碎片合成或每日盲盒的趣味玩法。
- **请输出**：
  - **换肤机制架构**：如何设计前端的卡牌渲染逻辑，使用户切换主题时，App 动态加载对应的图片资源路径（Assets Mapping）。
  - **抽卡概率引擎（Gacha Logic）**：使用代码（如 TypeScript/Python）写一个基础的抽卡算法。要求支持设置稀有度（如：大阿卡纳为 SSR，小阿卡纳为 SR/R），并包含每日免费抽卡次数、集齐牌组解锁专属皮肤的逻辑。

2. **核心业务代码实现**（给出关键的算法、数据转换、或者抽卡概率控制代码，语言可选择：TypeScript, Python 或 Go 之一）
3. **留存增长说明**（从产品经理角度简述该功能如何解决“有事才测”的低频痛点，提升用户日活 DAU）
